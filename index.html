PRESALES LEARNING GUIDE

SAP PLM System Integration
(PLMSI)

외부 PLM 시스템과 SAP S/4HANA 간의 양방향 제품 데이터 통합을 위한 SAP의 핵심 솔루션을 완벽하게 이해하고, Solution Advisory를 수행할 수 있는 역량을 갖추기 위한 학습 가이드입니다.

Scope Item: 6U2 Application Component: PLM-INT Latest Version: 3.0 (2024) Target: SAP S/4HANA Cloud & On-Premise
CHAPTER 01

Solution History

SAP PLM System Integration의 진화 과정과 현재까지의 발전 히스토리를 이해합니다.

2000년대 초반
SAP PLM (Product Lifecycle Management) 도입
SAP R/3에서 SAP PLM이 처음 소개됨. 자체 Engineering Record 기능으로 제품 개발 데이터를 관리하기 시작. 그러나 많은 제조 기업들은 이미 Siemens Teamcenter, PTC Windchill 등 전문 PLM 시스템을 사용하고 있었음.
2010년대 초반
PLM System Integration 1.0 for SAP ERP
외부 PLM 시스템과 SAP ERP 간의 통합 솔루션 첫 출시. 기존 ECTR(Engineering Control Center) 기반의 통합을 보완하여, 보다 구조화된 데이터 교환을 지원. SAP ERP 환경(ECC 6.0)에서 Material Master, BOM, Routing 정보의 교환을 가능하게 함.
2021년 (Q3)
PLM System Integration 2.0 for SAP S/4HANA
SAP S/4HANA 환경에 최적화된 2.0 버전 출시 (GA: 2021-10-25). REST API 기반 아키텍처로 전환, Meta Domain Model 도입. Siemens Teamcenter와의 통합을 공식 지원 시작. Maintenance 종료: 2025-12-31.
2022년
SAP Teamcenter Integration 2101 출시 및 종료
Siemens Teamcenter 전용 통합 버전. 빠르게 3.0으로 전환되며 2022-12-31에 Maintenance 종료.
2024년 3월
PLM System Integration 3.0 for SAP S/4HANA (Current)
최신 버전 (GA: 2024-03-25). SAP S/4HANA Cloud Public Edition 완전 지원, 4개 PLM 시스템 공식 지원 (Teamcenter, 3DEXPERIENCE, Windchill, Autodesk Vault). Advanced Change Management, Data Federation, Variant Configuration 등 고급 기능 대폭 강화. Maintenance 종료 예정: 2026-12-31.
현재 ~ 미래
Cloud-First 전략 및 확장
SAP S/4HANA Cloud Public Edition 중심의 전략 강화. Process Industry(Recipe), Quality Management 연계 확장. AI/ML 기반 자동 매핑 및 데이터 품질 향상 방향으로 발전 중.
핵심 포인트

PLMSI는 SAP ERP 시대부터 존재했으나, S/4HANA와 REST/JSON 기반 아키텍처로의 전환이 2.0부터 본격화되었습니다. 현재 3.0이 최신이며, SAP S/4HANA Cloud Public Edition을 완전히 지원하는 첫 번째 버전입니다.

CHAPTER 02

Core Concepts

PLM System Integration의 핵심 개념과 존재 이유를 이해합니다.

왜 PLMSI가 필요한가?

🏭

Engineering vs. Manufacturing Gap

설계팀은 Teamcenter/Windchill에서 CAD 설계를 하고, 생산팀은 SAP에서 MRP/PP를 실행합니다. 두 세계를 연결하는 것이 PLMSI의 존재 이유입니다.

🔄

Bi-directional Integration

단순한 단방향 데이터 전송이 아닌, 양방향 실시간 통합을 제공합니다. Engineering Change가 SAP에 반영되고, Manufacturing 피드백이 PLM에 전달됩니다.

📊

Single Source of Truth

제품 데이터의 일관성을 보장합니다. Engineering BOM과 Manufacturing BOM의 동기화, Change Management의 통합적 관리를 실현합니다.

Meta Domain Model - 핵심 아키텍처 개념

Meta Domain Model이란?

외부 PLM 시스템과 SAP S/4HANA 사이의 "공통 언어"입니다. PLM 시스템마다 데이터 모델이 다르지만, Meta Domain Model을 통해 표준화된 형태로 데이터를 교환합니다. REST Protocol과 JSON Format을 사용하며, 확장 가능한 구조입니다.

고객에게 설명할 때, "PLMSI는 단순한 인터페이스가 아니라, Engineering과 Manufacturing 프로세스를 하나의 디지털 스레드(Digital Thread)로 연결하는 프로세스 통합 솔루션"이라고 포지셔닝하세요.

Layer Technology Description
Protocol REST (HTTP/HTTPS) Stateless, scalable communication. 기존 RFC/BAPI 대비 현대적이고 유연한 방식

SAP에서 PLM 데이터 조회

SAP Fiori/GUI에서 PLM의 설계 도면, 3D 모델, 변경 이력 등을 실시간 확인. File Federation을 통한 문서 접근.

Advanced Structure Version

Version-managed BOM의 데이터 연합을 통해 설계 버전과 생산 버전의 실시간 비교 및 추적.

Process Flows

SAP Best Practices에 정의된 PLMSI의 핵심 프로세스 플로우를 이해합니다.

Scope Item 6U2 - Process Flow Overview

PLMSI Scope Item (6U2)은 SAP Best Practices for SAP S/4HANA Cloud Public Edition에 포함되어 있으며, NPDI(New Product Development & Introduction) 시나리오를 기반으로 합니다.

📝

Variant 1: Product Engineering

순수 설계 데이터 전송 시나리오. Design Engineer가 주도.

  • Create Engineering Data
  • Create/Change Engineering BOM
  • Create Change Notice
  • Implement Design Changes
  • Create BOM Revision
  • Release Engineering Change to SAP
  • Handover Engineering Data to Manufacturing
  • Complete Change
⚙

Variant 2: Product & Manufacturing Engineering

설계 + 생산 엔지니어링 통합 시나리오. 두 Role이 협업.

  • Create Engineering Data
  • Create Material Master
  • Create BOM Revision
  • Create Production Operation List
  • Create Routing
  • Assign Part & Workarea to Operation
  • Tool Assignment
  • Create Change Notice
  • Implement Design Changes
  • Release Mfg Change Notice to SAP
  • Complete Change

Supported Business Scenarios

Make-to-Stock (MTS)

표준 제품의 대량 생산. Engineering BOM이 확정된 후 Manufacturing BOM으로 릴리스.

Make-to-Order (MTO)

주문 기반 생산. Customer Order Exchange를 통해 주문별 BOM 생성.

Configure-to-Order (CTO)

설정 가능한 제품. Variant Configuration과 연동하여 고객 요구사항에 맞는 제품 구성.

Positioning Strategy

Customer Scenario-Based Positioning

Scenario: Greenfield S/4HANA Implementation

고객이 SAP S/4HANA를 처음 구축하면서 기존 PLM 시스템과의 통합이 필요한 경우.

Positioning Approach
  • "Clean Core" 전략과 일치 - 표준 통합으로 S/4HANA의 혁신 속도 유지
  • Scope Item 6U2 활성화로 빠른 구현 가능 (Best Practice 기반)
  • 기존 PLM 투자를 보호하면서 SAP 통합의 이점을 최대화
  • 초기 Scope: Material + BOM + Change Management → 점진적 확장

Scenario: Cloud Transformation

SAP S/4HANA Cloud Public Edition으로 전환하면서 PLM 통합을 구현하는 경우.

Positioning Approach

CHAPTER 10

Objection Handling

고객이 자주 제기하는 이의와 그에 대한 대응 방법을 습득합니다.

"우리는 이미 커스텀 인터페이스가 있습니다"

Response: 커스텀 인터페이스는 S/4HANA 업그레이드 시마다 재검증/재개발이 필요합니다. 특히 Cloud Public Edition에서는 커스텀 인터페이스 구현이 제한됩니다. 표준 솔루션은 SAP의 릴리스 사이클에 맞춰 자동 업데이트되며, 11개 도메인의 시나리오가 이미 구현되어 있어 추가 개발 비용을 절감합니다.

"Add-on 비용이 추가됩니다"

Response: TCO 관점에서 비교하세요. 커스텀 개발+유지보수+업그레이드 비용 vs. 표준 라이선스+유지보수 비용. 일반적으로 3년 이내에 손익분기점에 도달합니다. 또한 표준 솔루션의 기능 확장은 추가 비용 없이 업데이트로 제공됩니다.

"우리 프로세스가 너무 특수합니다"

Response: PLMSI는 BRFplus 기반의 Advanced Field Control, Table-like Structure로의 확장, Custom Mapping Rules 등을 통해 고객별 요구사항을 수용합니다. 또한 Meta Domain Model 자체가 확장 가능하도록 설계되었습니다. 먼저 Fit-to-Standard 분석을 통해 Gap을 정확히 파악하는 것을 권장합니다.

"Teamcenter에 자체 SAP 연동 기능이 있는데?"

Response: Siemens가 제공하는 자체 연동과 SAP PLMSI는 접근 방식이 다릅니다. PLMSI는 SAP 측에서 인증/유지보수하는 솔루션으로, SAP 업그레이드와의 호환성이 보장됩니다. 또한 SAP의 Deep Integration(Change Management, Variant Configuration, Quality Management 등)을 완전하게 지원하며, SAP Support를 직접 받을 수 있습니다.

"Cloud Public Edition의 제약이 걱정됩니다"

Response: PLMSI 3.0은 Cloud Public Edition을 위해 설계되었습니다. Scope Item 6U2로 활성화되며, 3rd party integration으로 분류되어 별도 요청을 통해 활성화 가능합니다. 표준 프로세스 범위 내에서 대부분의 통합 요구사항을 충족하며, Clean Core 원칙을 유지하면서 PLM 통합을 달성합니다.

"구현에 너무 오래 걸리지 않나요?"

Response: Best Practice 기반의 사전 구성된 프로세스가 포함되어 있어, 일반적인 초기 구현(Material + BOM + Basic Change Mgmt)은 3-4개월 내에 Go-Live 가능합니다. 이후 점진적으로 시나리오를 확장하는 Phased Approach를 권장합니다.

Knowledge Check

학습 내용을 점검하고 Presales 대화에서 활용할 수 있는지 확인합니다.

Q1. PLMSI에서 외부 PLM 시스템과 SAP S/4HANA 사이의 데이터 교환에 사용되는 핵심 개념은?
Correct! Meta Domain Model은 PLM 시스템과 SAP 사이의 공통 데이터 모델로, REST/JSON을 통해 교환됩니다.
Q2. PLMSI가 공식 지원하는 외부 PLM 시스템이 아닌 것은?
Correct! Arena PLM은 PLMSI에서 공식 지원하지 않습니다. 공식 지원: Teamcenter, 3DEXPERIENCE, Windchill, Autodesk Vault.
Q3. PLMSI의 현재 최신 버전(for S/4HANA)과 GA 일자는?
Correct! PLM System Integration 3.0 for SAP S/4HANA는 2024년 3월 25일 GA되었으며, 2026년 12월 31일까지 유지보수됩니다.
Q4. 3DEXPERIENCE용 Add-On을 제공하는 파트너사는?
Correct! CENIT (cenit.com)이 3DEXPERIENCE용 Add-On을 제공합니다. RIESS=Windchill, CIDEON=Autodesk Vault, SAP Direct=Teamcenter.
Q5. PLMSI Scope Item의 식별 코드는?
Correct! 6U2가 PLM System Integration의 Scope Item 코드입니다. PLM-INT는 Application Component 코드, 71U는 Building Block 코드입니다.
Q6. PLMSI가 속하는 Business Process Group은?
Correct! PLM System Integration은 "R&D and Engineering" Business Process Group에 속합니다.
